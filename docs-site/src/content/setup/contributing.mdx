# Contributing

## Scope

This repo now has four active plugins:

- `eneo-core`
- `eneo-standards`
- `eneo-findings`
- `eneo-review`

If you add or change functionality, keep it within one of those surfaces unless there is a strong reason to introduce a new plugin.

## Versioning

For this marketplace, plugin update detection is managed in `.claude-plugin/marketplace.json` because the plugins are listed as relative-path entries there. Bump the marketplace entry version when you want installed users to receive an update.

## Local development

Use `--plugin-dir` first:

```bash
claude \
  --plugin-dir ./plugins/eneo-core \
  --plugin-dir ./plugins/eneo-standards \
  --plugin-dir ./plugins/eneo-findings \
  --plugin-dir ./plugins/eneo-review
```

Then test:

```bash
/eneo-core:eneo-doctor
```

## Keep the common path light

The harness is designed to keep routine work simple. Prefer:

- bounded commands
- deterministic helpers
- clear docs

Avoid adding new coordination layers or required external tooling unless the gain is obvious.
