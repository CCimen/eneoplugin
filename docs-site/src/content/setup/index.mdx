# Installation

## Prerequisites

- Claude Code installed
- Git access to this repo
- `gh` authenticated if you want findings or PR flows

## Add the marketplace

```bash
/plugin marketplace add CCimen/eneoplugin
```

## Install the current plugins

```bash
/plugin install eneo-core@eneoplugin
/plugin install eneo-standards@eneoplugin
/plugin install eneo-findings@eneoplugin
/plugin install eneo-review@eneoplugin   # optional
/reload-plugins
```

## First command to run

```bash
/eneo-core:eneo-doctor
```

That verifies:

- environment mode
- container routing
- required tools
- current task state
- ratchet baseline presence

## Notes

- `eneo-review` is optional because it is only used on risky changes
- `eneo-findings` needs GitHub project access if you plan to create findings
- `--plugin-dir` is the best way to iterate locally during development
