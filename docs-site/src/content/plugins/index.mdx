# Plugins Overview

The marketplace now ships four plugins:

| Plugin | Purpose |
|--------|---------|
| [eneo-core](/plugins/eneo-core) | Main `/eneo-*` workflow, subagents, and domain skills |
| [eneo-standards](/plugins/eneo-standards) | Hooks, validators, helper binaries, and status line |
| [eneo-findings](/plugins/eneo-findings) | Findings workflow and learning extraction |
| [eneo-review](/plugins/eneo-review) | Trigger-gated external review agents |

## How they fit together

- `eneo-core` is the command surface you use most often
- `eneo-standards` is the runtime safety layer
- `eneo-findings` handles backlog capture outside the current task
- `eneo-review` stays quiet until `/eneo-verify` decides a change is risky enough to justify extra review

This is intentional. The harness is designed to keep the common path lightweight and only pay extra complexity when risk or scope actually justify it.
