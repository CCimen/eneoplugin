# Finding

Track findings, bugs, and ideas as GitHub issues on the [eneo project board](https://github.com/orgs/eneo-ai/projects/1).

## Why

When working on a task, you often discover things that should be fixed but are out of scope. Instead of losing track of them or breaking your flow, use `/finding` to quickly log them as well-structured issues.

Issues are written with AI-actionable sections so that Claude (or another AI) can pick them up and work on them later without needing additional context.

## Setup

> New to eneoplugin? Start with the [Installation guide](/setup) to add the marketplace first.

Install the plugin and add the GitHub project scope:

```bash
/plugin install finding@eneoplugin
gh auth refresh -s project -h github.com
```

Verify access:

```bash
gh project view 1 --owner eneo-ai
```

## Commands

### Create a finding

```
/finding
/finding <description>
```

Creates a new GitHub issue from a finding. Claude will:

1. Gather context (ask if needed, skip if context is clear from the conversation)
2. Draft the issue with structured sections
3. Show you the draft for confirmation
4. Create the issue and add it to the project board

**Issue structure:**

```markdown
## Problem
What's wrong or missing

## Context
- Area: which part of the codebase
- Discovered while: what you were working on
- Related files: specific file paths

## Suggested Solution
Concrete approach for fixing/implementing

## Acceptance Criteria
- [ ] What "done" looks like
```

### List open items

```
/finding list
```

Shows all **Todo** and **In Progress** items on the project board in a table format.

### Grab an issue

```
/finding grab <number>
```

Pick up an issue to start working on it. Claude will:

1. Fetch and display the issue details
2. Offer to create a feature branch and start working
3. Move the item to "In Progress" on the board
4. Assign you to the issue

## Labels

Available labels: `bug`, `enhancement`, `documentation`, `question`

## Example

During a conversation about WebSocket handling:

> "Hmm, I notice the reconnection logic doesn't handle token refresh. Let's not fix it now though."
>
> `/finding`
>
> Claude drafts: **"Fix WebSocket reconnection to handle expired auth tokens"** with full context and suggested solution.
