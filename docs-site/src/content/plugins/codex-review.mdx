# Codex Review

Multi-persona peer review via OpenAI Codex CLI. Provides an independent second opinion on architecture and code quality.

## Why

Having a second AI review your code catches blind spots and validates architectural decisions. Codex Review acts as a structured peer reviewer with multiple specialized personas.

## Prerequisites

```bash
codex --version  # Must be installed
```

## Setup

```bash
/plugin install codex-review@eneoplugin
```

## Usage

```
/codex-review
```

Claude auto-invokes this for plan reviews, mid-implementation code reviews, and consultations. You can also trigger it manually.

## Operating modes

### Plan Review

Verifies architecture before implementation. Runs approximately 3 iterations to validate the approach.

### Code Review

Reviews `git diff` mid-implementation for quality, maintainability, and testability.

### Consultation

Answers specific questions when Claude is unsure about an approach.

## Personas

Each review includes perspectives from four specialized personas:

| Persona | Focus |
|---------|-------|
| **Solution Architect** | Overall architecture and design (always active) |
| **API Consumer** | Endpoint design, contracts, usability |
| **Security Reviewer** | Auth, permissions, data handling |
| **Performance Analyst** | Query efficiency, hot paths, scalability |

## Pipeline

Codex Review is the second step in the review pipeline:

```
Claude (implements) → Codex (primary review) → Gemini (cross-check) → Claude (synthesis)
```
